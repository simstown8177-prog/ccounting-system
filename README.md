# 샵앤샵 평택1호점 재고 관리

샵앤샵 형태의 요식업 매장을 위한 간편형 재고 관리 화면입니다. 메인에서 카테고리를 선택하고,
해당 카테고리 계정으로 로그인한 뒤 독립된 재고를 관리합니다.

## 포함 기능

- 메인 페이지에서 5개 카테고리 선택
- 카테고리별 독립 로그인
- 비밀번호 해시 저장
- 관리자 전용 사용자 추가
- 카테고리별 독립 재고 관리
- 카테고리별 거래처 관리
- 발주 등록 및 입고 처리
- 수동 마감 차감
- 영수증 이미지 업로드 기반 차감
- 일자별 마감 리포트
- 기준 재고 이하 화면 경고, 브라우저 알림, 소리 알람
- 브라우저 푸시 알림 구독
- 카카오 발주 알림용 메시지 생성
- 카카오 발주 전송 시뮬레이션
- OCR 텍스트 기반 품목 매칭 미리보기

## 실행 방법

루트 디렉터리에서 아래 명령으로 서버를 실행합니다.

```bash
npm start
```

그 다음 브라우저에서 `http://localhost:3000`으로 접속하면 됩니다.

운영 데이터는 `DATABASE_URL`이 있으면 외부 Postgres에 저장되고, 없으면 로컬
[data/app.db](/home/user/ccounting-system/data/app.db)에 저장됩니다.

기존 [data/store.json](/home/user/ccounting-system/data/store.json)은 최초 DB 마이그레이션용
백업 소스로만 사용됩니다.

환경변수 예시는 [.env.example](/home/user/ccounting-system/.env.example)에 정리했습니다.

- `HOST`: 기본값 `0.0.0.0`
- `PORT`: 기본값 `3000`
- `DB_PATH`: 로컬 SQLite DB 파일 경로
- `DATABASE_URL`: Render free 배포용 외부 Postgres 연결 문자열

브라우저 푸시 알림은 `localhost`에서는 바로 테스트할 수 있고, 외부 배포 환경에서는 보통
`https`가 필요합니다.

## 기본 로그인

- 사용자 `manager` / 비밀번호 `1234`
- 사용자 `staff` / 비밀번호 `1111`

각 계정은 카테고리별로 분리되어 있습니다.

- `manager`는 사용자 추가 같은 관리자 기능을 사용할 수 있습니다.
- `staff`는 일반 운영용 계정입니다.

## 영수증 업로드 차감 방식

현재 버전은 OCR 자동 인식이 아닌 `영수증 이미지 업로드 + 차감 품목 직접 선택` 방식입니다.
즉, 영수증 파일을 첨부하고 어떤 품목이 얼마만큼 차감되었는지 바로 입력해 재고를 줄입니다.

추가로 영수증 텍스트를 붙여 넣으면 등록 품목명과 일치하는 항목을 미리 추천하는 OCR
미리보기 기능이 포함되어 있습니다.

## 카카오 연동 범위

현재는 로컬 서버 기반 버전이며 카카오로 실제 자동 전송은 하지 않고, 발주 알림 메시지를
생성, 복사, 전송 시뮬레이션할 수 있게 구현했습니다.

실제 자동 발송을 하려면 별도 서버, 인증키, 카카오 비즈니스 API 연동이 추가로 필요합니다.

## 브라우저 푸시 알림

로그인 후 상단의 `브라우저 푸시 켜기` 버튼을 누르면 현재 카테고리에 대해 푸시 구독이 저장됩니다.
이후 기준 재고 이하 상태가 새로 발생하면 서버가 해당 카테고리 구독 브라우저로 푸시를 보냅니다.

## 배포 메모

- Render 무료 플랜에서는 `Persistent Disk`를 못 쓰므로 `DATABASE_URL`로 외부 Postgres를 연결해야 합니다.
- Render 유료 플랜이나 로컬 테스트에서는 SQLite fallback을 계속 쓸 수 있습니다.
- 브라우저 푸시는 운영 환경에서 보통 `HTTPS`가 필요합니다.
- 현재 저장소는 `Postgres 우선 + SQLite fallback` 기준으로 배포 가능한 상태입니다.
